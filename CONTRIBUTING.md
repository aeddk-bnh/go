# Contributing

Thanks for contributing! Please follow these guidelines to keep the project healthy and high-quality.

## How to contribute
1. Fork the repository and create a branch named `feature/your-short-desc` or `fix/issue-123`.
2. Write tests for new features/bug fixes.
3. Follow the coding and commit message standards below.
4. Open a Pull Request against `main` and describe the change, tests performed, and any migration notes.

## Coding standards
- Target C++17 or later.
- Use `clang-format` configuration in repository (project uses 2-space indent, K&R style).
- Add unit tests for behavioral changes. Use GoogleTest for new tests.

## Code reviews
- Each PR must have at least one approving review.
- Fix CI issues before merging.

## Issue reporting
- Provide steps to reproduce, expected vs actual behavior, and minimal reproducer if possible.
