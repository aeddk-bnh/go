# Go Game (C++)

A C++ implementation of the board game Go. This repository contains the core engine, a console UI, and scaffolding for GUI, AI, and network play.

## Quick start

Requirements: C++17, CMake, a C++ compiler (gcc/clang/Visual Studio).

```bash
# build (out-of-source)
mkdir build && cd build
cmake ..
cmake --build . --config Release

# run tests
ctest --output-on-failure
```

## Quick build (recommended)

On Windows (PowerShell):

```powershell
# install Conan (if not installed)
python -m pip install --user conan

# run helper script (uses Conan if available)
.\scripts\build.ps1 -BuildDir build -Config Release

# run tests
ctest --test-dir build/tests --output-on-failure
```

On Linux/macOS:

```bash
python -m pip install --user conan
./scripts/build.sh build Release
ctest --test-dir build/tests --output-on-failure
```

Notes:
- By default benchmarks are disabled for faster builds. Enable them with `-DBUILD_BENCHMARKS=ON`.
- The helper scripts detect a `conan_toolchain.cmake` generated by `conan install` and will use it automatically.

## Developer setup
1. Install prerequisites: CMake, a C++17-capable compiler, and optional tools: `clang-format`, `clang-tidy`.
2. Build:
```bash
mkdir build && cd build
cmake ..
cmake --build .
```
3. Run tests:
```bash
ctest --output-on-failure
```
4. Run formatting and lint checks:
```bash
clang-format -style=file -i $(git ls-files '*.cpp' '*.hpp' '*.h')
clang-tidy <file> -- -Iinclude
```

### Provisioning deps and dev tools with Conan (recommended)

Prefer using Conan to avoid committing large vendor binaries into the repo. Example:

```powershell
# install conan if needed
python -m pip install --user conan

# install requirements + tool_requires (gtest, benchmark, cppcheck, ninja)
conan install . --output-folder build --build missing

# configure and build
cmake -S . -B build -G Ninja -DCMAKE_TOOLCHAIN_FILE=build\conan_toolchain.cmake
cmake --build build -- -j
```

After verifying Conan provisioning, you can remove or git-ignore the vendored `/tools` contents to shrink the repository.
